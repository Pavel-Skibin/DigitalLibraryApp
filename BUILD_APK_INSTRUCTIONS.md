# Инструкция по сборке APK для установки на Android устройство

## Шаг 1: Узнайте IP адрес вашего компьютера

### Windows:
1. Откройте командную строку (Win + R → `cmd`)
2. Введите: `ipconfig`
3. Найдите "IPv4 Address" в разделе вашего сетевого адаптера
4. Запишите IP (например, `192.168.1.100`)

### Linux/Mac:
1. Откройте терминал
2. Введите: `ifconfig` или `ip addr`
3. Найдите IP адрес (обычно начинается с `192.168.` или `10.`)

## Шаг 2: Измените BASE_URL в коде

Откройте файл: `app/src/main/java/org/nahap/library/di/NetworkModule.kt`

Измените строку:
```kotlin
private const val BASE_URL = "http://192.168.1.XXX:8080/"
```

Замените `XXX` на ваш IP адрес, например:
```kotlin
private const val BASE_URL = "http://192.168.1.100:8080/"
```

## Шаг 3: Убедитесь, что сервер доступен

1. Убедитесь, что ваш Spring Boot сервер запущен на порту 8080
2. Проверьте, что сервер доступен по IP адресу (не только localhost):
   - Откройте браузер на компьютере
   - Перейдите по адресу: `http://ВАШ_IP:8080/api/books/1/fb2`
   - Должен вернуться FB2 контент

## Шаг 4: Соберите APK через Android Studio

### Вариант A: Через меню (рекомендуется)

1. В Android Studio выберите: **Build → Generate Signed Bundle / APK**
2. Выберите **APK** → **Next**
3. Выберите **debug** keystore (для тестирования) или создайте новый:
   - Нажмите **Create new...**
   - Заполните данные (можно любые для тестирования)
   - Сохраните keystore файл
4. Выберите **release** build variant → **Finish**
5. APK будет создан в: `app/build/outputs/apk/release/app-release.apk`

### Вариант B: Через Build Variants

1. В Android Studio: **Build → Select Build Variant**
2. Выберите **release** для модуля `app`
3. Выберите: **Build → Build Bundle(s) / APK(s) → Build APK(s)**
4. APK будет создан в: `app/build/outputs/apk/release/app-release.apk`

## Шаг 5: Соберите APK через командную строку

Откройте терминал в корне проекта и выполните:

```bash
# Windows
gradlew.bat assembleRelease

# Linux/Mac
./gradlew assembleRelease
```

APK будет создан в: `app/build/outputs/apk/release/app-release.apk`

## Шаг 6: Установите APK на устройство

### Способ 1: Через USB (ADB)

1. Включите **Отладка по USB** на Android устройстве:
   - Настройки → О телефоне → Нажмите 7 раз на "Номер сборки"
   - Настройки → Для разработчиков → Включите "Отладка по USB"
2. Подключите устройство к компьютеру через USB
3. В терминале выполните:
   ```bash
   adb install app/build/outputs/apk/release/app-release.apk
   ```

### Способ 2: Через файловый менеджер

1. Скопируйте `app-release.apk` на Android устройство (через USB, email, облако и т.д.)
2. На устройстве откройте файловый менеджер
3. Найдите APK файл и нажмите на него
4. Разрешите установку из неизвестных источников (если потребуется)
5. Нажмите "Установить"

## Шаг 7: Проверьте подключение

1. Убедитесь, что Android устройство и компьютер в одной Wi-Fi сети
2. Запустите приложение на устройстве
3. Проверьте, что оно может подключиться к серверу

## Важные замечания

⚠️ **Для продакшена:**
- Создайте собственный keystore для подписи APK
- Используйте реальный домен вместо IP адреса
- Включите ProGuard/R8 для уменьшения размера APK
- Настройте HTTPS вместо HTTP

🔒 **Безопасность:**
- Не используйте `usesCleartextTraffic="true"` в продакшене
- Используйте HTTPS для всех API запросов

📱 **Размер APK:**
- Debug APK обычно больше (5-10 MB)
- Release APK меньше (2-5 MB) после включения minification
